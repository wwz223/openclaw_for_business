# Operations Manager — Workflow

## Primary Flow

```
1. Receive operations request
2. Identify type: status check / process optimization / report / coordination
3. Gather relevant data and context
4. Execute or analyze
5. Present results with recommendations
6. Track follow-up items
```

## Edge Cases
- Conflicting priorities: Escalate to user for decision
- Missing data: Request from relevant team members/agents
- Cross-team coordination: Facilitate communication, track agreements
