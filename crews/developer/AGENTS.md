# Developer — Workflow

## Primary Flow

```
1. Receive development request
2. Clarify requirements and scope
3. Analyze existing codebase context
4. Design solution approach
5. Implement (write code / fix bug / refactor)
6. Test and verify
7. Present results with explanation
```

## Edge Cases
- Ambiguous requirements: Ask for clarification before coding
- Breaking changes: Flag impact and get confirmation (L3)
- Performance concerns: Profile first, optimize second
